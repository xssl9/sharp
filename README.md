# Sharp — Jarvis-подобный AI-агент для терминала

TUI-ассистент в стиле cava: аудио-визуализатор в терминале, диалог через **Gemini API**,
**голосовой ответ прямо из Gemini** по одному API-ключу (нативный TTS, без сторонних сервисов),
голосовой ввод с микрофона и управление Linux-системой.

## Возможности
- 🎙️ Голосовой ввод (микрофон → Google STT) — по `Ctrl+K`
- 🧠 Ответы Gemini (`gemini-3-flash-preview` с фолбэком на другие модели)
- 🔊 Озвучка ответов нативным Gemini TTS (PCM 24 кГц) — «сразу вывод в звуке»
- 📊 Cava-подобный визуализатор, который двигается под голос ответа
- 🖥️ Команды под Linux: медиа (`playerctl`), громкость (`pactl`), сайты/приложения
  (`xdg-open`), Steam-игры, чтение/список файлов

## Установка
```bash
cd ~/sharp
python3 -m venv .venv
.venv/bin/pip install -e .
```

## Ключ Gemini
В `~/.config/sharp/config.toml`:
```toml
api_key = "ТВОЙ_КЛЮЧ"
voice = "Charon"          # 30 голосов: Kore, Puck, Zephyr, Fenrir, Aoede, ...
mic_sensitivity = 1.0     # 0.25–1.25; выше — чувствительнее
chat_model = "gemini-3-flash-preview"
allow_shell_commands = false  # безопасное значение по умолчанию
```
или `export GEMINI_API_KEY=…`

После первого изменения настроек старый `api_key` автоматически переносится в
`~/.config/sharp/credentials.toml`. Ключ из переменной окружения на диск не записывается.

Чтобы Шарп мог выполнять голосовые команды терминала и видеть их stdout/stderr,
установите `allow_shell_commands = true`. Открытие обычных приложений работает
независимо от этой настройки. Установка пакета выполняется отдельным инструментом:
Шарп сначала спрашивает подтверждение, затем открывает видимый терминал с
`sudo pacman -S --needed <пакет>`.

Голосовые примеры:

- «Шарп, выполни `uname -a` и скажи, что вывело» — Шарп получает код возврата,
  stdout и stderr команды и пересказывает результат.
- «Шарп, открой Dolphin» — приложение запускается даже при выключенных произвольных
  terminal-командах.
- «Шарп, установи Firefox через pacman» — Шарп называет пакет и спрашивает
  подтверждение; после голосового «да» открывает установку в `kitty`.

```bash
sharp                 # или: .venv/bin/python -m sharp
```

### Горячие клавиши
| Клавиша  | Действие        |
|----------|-----------------|
| `Enter`  | отправить текст |
| `Ctrl+K` | микрофон        |
| `Ctrl+L` | очистить чат    |
| `Ctrl+C` | выход           |

### Голосовое управление — основной способ

Микрофон слушает wake-word: начинайте команду с имени **«Шарп»**, например
«Шарп, включи Мою волну». Речь без обращения к Шарпу игнорируется и не отправляется
в Gemini. Можно сказать только «Шарп»: после мгновенного звукового подтверждения
откроется 12-секундное
окно, в котором следующую команду уже не нужно начинать с имени. При задержке
соединения 200 мс и выше Live остаётся активным, но сначала загружает весь
голосовой PCM-ответ и только затем воспроизводит его без заиканий. Если сеть
просядет во время разговора, буферный режим включится автоматически.

Статус справа снизу показывает этап: `HEARING` — голос уже захвачен,
`RECOGNIZING` — распознаётся слово, `LISTENING` и короткий сигнал — можно говорить
следующую команду без повторения имени.

- «Шарп, покажи список сессий» → Шарп голосом уточнит: Codex или Claude Code
- «Шарп, передай Кодексу исправить визуализатор» → покажет сессии и спросит номер
- «Первая сессия», «вторая», «новая» или «отмена» → завершит голосовой выбор
- «Включи Мою волну», «пауза», «продолжай», «следующий/предыдущий трек»
- «Сделай Яндекс Музыку громче/тише»
- «Включи исполнителя Кино в Яндекс Музыке»
- «Запомни …», «что ты помнишь», «очисти память», «смени голос на Kore»

После команды Яндекс Музыке короткие продолжения вроде «ещё тише» или «следующий»
тоже относятся к ней. Печатать команды не требуется.

### Необязательные текстовые дубли
- `/voice Kore` — сменить голос
- `/mute` — вкл/выкл озвучку
- `/clear` — очистить историю
- `/sessions codex` или `/sessions claude` — показать последние сессии
- `/agent codex <задача>` — выбрать сессию и делегировать задачу
- `/ym wave` — открыть или продолжить Мою волну
- `/ym play|pause|next|prev|volumeup|volumedown` — управление Яндекс Музыкой
- `/ym artist <имя>` — открыть исполнителя в Яндекс Музыке

Можно говорить естественно: «делегируй Codex исправить тесты» — Шарп покажет
последние сессии и попросит выбрать номер; «включи Мою волну», «следующий трек
в Яндекс Музыке», «сделай Яндекс Музыку тише», «включи исполнителя Кино».

## Проверка озвучки отдельно
```bash
.venv/bin/python -m sharp.tts "Привет, сэр. Я Шарп."
# → out.wav (24 кГц) + воспроизведение
```

## Архитектура
```
микрофон → stt.py → assistant.py (Gemini chat + команды) → gemini.synth (TTS)
         → audio.py (проигрывание PCM + уровни) → tui/visualizer.py (бары)
```
- `gemini.py` — единый клиент: `chat()` и `synth()` по одному ключу
- `commands.py` — порт логики из старой Electron-версии на Linux-инструменты
- `config.py` — конфиг из toml + env
- `storage.py` — атомарная запись настроек, истории и памяти

TUI выполнен в минимальном чёрно-белом стиле: Cava-подобные раздельные полосы
занимают весь терминал, последние реплики выводятся коротким оверлеем, а ввод и
состояние сессии помещаются в единственную строку снизу.
